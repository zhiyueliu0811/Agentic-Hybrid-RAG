## 三、每个版本的细节

### v1：概念验证

- **数据构造**：从 SFT 模型 300 queries × 4 temperatures 生成候选，judge 打分，自然分差选 pair + 2 种合成（去引用、截断）
- **核心问题**：`top_k=1` 导致 85% query 的 4 个温度产出完全相同答案
- **局限**：300 对太少，对比维度单一

### v2：扩展合成类型

- **数据构造**：复用 v1 候选（低多样性），扩到 6 种合成类型：自然分差、去引用、截断、假引用、拒答、冗余引用
- **关键发现**：自然分差仅 13 对（2%），合成对占 98%
- **致命缺陷**：拒答对方向是反的！`chosen=正常回答 > rejected="无答案"`，训练模型「任何回答都比拒答好」
- **局限**：幻觉率恶化，候选多样性没修复

### v3：修复候选多样性（关键突破）

- **数据构造**：修复 `top_k=1` bug，用 4 种采样配置 `[T=0.5, 1.0, 1.2, 1.5]` 重新生成候选 → judge 重新打分 → 构造 1344 对
- **自然分差从 13 → 200（+1438%）**
- **核心改进**：引用质量显著提升，答案更简洁
- **局限**：拒答对仍然反向（chosen=回答 > rejected=拒答），幻觉率未改善，多了 3 条错误拒答（FP=3, 6%）

### v4：RL 飞轮修正拒答

- **数据构造**：
  1. 合成 95 条无关中文问题（应拒答场景）
  2. 用 v3 跑一遍，发现 95/95 都回答了（0% 拒答率，验证了问题）
  3. 构造修正对：`chosen=拒答 / rejected=v3的错误答案`
  4. 合并到 v3 的 1344 对 → 1439 对
- **训练**：从 v3 checkpoint 继续，1 epoch，lr 减半（2e-6）
- **效果**：错误拒答从 3 条降到 0 条，幻觉率基本持平
- **局限**：95 对拒答只占 6.6%，信号太弱

## 四、数据构造方法全集

### 4.1 候选生成 (`src/rl/generate_candidates.py`)

```python
# Phase 6.2 修复后
SAMPLING_CONFIGS = [
    {"temperature": 0.5, "top_p": 0.9},              # 保守
    {"temperature": 1.0, "top_p": 0.95, "top_k": 50}, # 适中
    {"temperature": 1.2, "top_p": 0.85},              # 激进
    {"temperature": 1.5, "top_p": 0.80},              # 高多样性
]
```

- 300 queries × 4 configs = 1200 候选
- 关键修复：去掉 `top_k=1`（强制贪婪），让 temperature 真正生效
- 效果：多样性从 15% → 93%

### 4.2 Judge 打分 (`src/rl/score_candidates.py` + `reward.py`)

- Judge: Qwen2.5-14B-Instruct-AWQ (vLLM :8001)
- 对每个候选做 CitationVerifier 校验
- 奖励函数：60% citation_support_rate + 25% unsupported_penalty + 长度/拒答辅助分

### 4.3 偏好对构造 (`src/rl/build_preference_pairs.py`)

6 种构造方法：

| 类型 | 方法 | 教什么 | 占比(v3) |
|------|------|--------|----------|
| 自然分差 | reward 最高 vs 最低 | 综合质量 | 15% |
| 去引用 | chosen 带【1】, rejected 移除 | 引用格式 | 20% |
| 假引用 | 真实 ID → 不存在的 ID【99】 | 引用要真实 | 20% |
| 冗余引用 | 精确引 1 条 → 引 3 条无关 | 引用要精准 | 20% |
| 截断 | chosen 完整, rejected 只留前半 | 答案要完整 | 4% |
| **拒答** | **chosen 正常答, rejected="无答案"** | **⚠️ 反向训练！** | 20% |

> 🔴 关键问题：拒答对方向是反的。chosen 永远是正常答案，教模型「回答 > 拒答」。

### 4.4 拒答修正对 (`src/rl/flywheel_refusal.py`)

```python
# 正确方向
chosen   = "抱歉，我是Model 3用户手册问答助手，无法回答这个问题。"
rejected = v3 对无关问题瞎编的答案（"红烧肉的做法是..."）
```

- 合成 95 条无关中文问题（日常生活、娱乐、通用、其他领域、无意义闲聊）
- 用当前模型跑 → 找出幻觉 → 构造修正对

## 五、关键发现

### 5.1 候选多样性是最重要的杠杆

```
top_k=1 (贪婪) → 85% 重复 → 自然分差 13 对 → Agentic RAG 0.8450
top_k 移除      → 93% 多样 → 自然分差 200 对 → Agentic RAG 0.8765
```

### 5.2 合成偏好对的信号方向决定一切

- 引用类 4 种（去引用/假引用/冗余/截断）+ 自然分差 = 教引用质量 ✅
- 拒答对方向反了（chosen=回答 > rejected=拒答）= 教模型永远回答 ❌

### 5.3 评测指标需要和行为对齐

- 旧 `is_refused` 只匹配 `"无答案"` 字面 → v2 礼貌拒答被误判为幻觉
- 改进版匹配 9 种拒答模式才反映真实行为

### 5.4 RL 飞轮可行但需要迭代

- v3 → 发现 0% 拒答 → 构造 95 修正对 → v4 消除错误拒答
- 但 6.6% 的拒答对占比太低，需要继续飞轮

- 必须 `VLLM_USE_V1=0`（V1 引擎 AWQ 编译失败）
